Functions

  • Defining functions
    • function in javaScript considered to "first-class objects"
    • Function x(){}
    • Function expressions anonymous function set to variable- has semincolon afer }
    • each function has "name" property
    • Function constructor new Function();
      • Not recommended to declare functions in this way as there are numerous problems associated with placing the function body inside a string.

  • Invoking a function
    • Invoking a function is to run the code inside the function’s body.
    • To invoke a function, simply enter its name, followed by parentheses.
    • functions helps to Keep Your Code DRY
      • Don’t Repeat Yourself, or DRY, is a principle of programming that specifies that every part of a program should only be written once.

  • Return values
    • All functions return a value, which can be specified using the return statement, which comes after the return keyword.
    • A function that doesn’t explicitly return anything will return undefined by default.

  • Parameters and arguments
    • Parameters and arguments are terms that are often used interchangeably to represent values provided for the function as an input.
    • Variable Numbers of Arguments
      • Every function has a special variable called arguments. This is an array-like object that contains every argument passed to the function when it is invoked.
    • Improved Mean Function
      • We can use a rest parameter to improve our mean() function so it accepts any number of values.
      • This collects all the arguments that are entered into an array called values.
    • Default Parameters
      • These are values that will be used by the function if no arguments are provided when it is invoked.
    • Arrow Functions
      • Arrow functions make perfect candidates for short, anonymous functions.
      • Single parameters don't need putting into parentheses.
      • The body of the function doesn't need placing inside a block if it's only one line.
      • The return keyword isn't required if the return statement is the only statement in the body of the function.
      • They don't bind their own value of this to the function.

  • Function Hoisting
    • Functions that are defined using a function declaration are automatically hoisted, meaning they can be invoked before they have been defined.

  • Variable Hoisting
    • var keyword hoists variable but not variable assignment.
    • something to consider when deciding how to declare functions.

  • Callbacks
    • A function passed as a parameter to another function
    • when passed as argument no parenthesis are used
    • anonymous function can also be passed as argument

  • Array Iterators
    • forEach()
    • map()
    • reduce()
    • Filter()
    • Chaining Iterators Together
      • The various iterator functions can be used in combination to create some powerful transformations of data stored in arrays.
