Events

  • Introduction to events
    • Event listeners in JavaScript work like setting a notification to alert you when something happens.
    • The following code can be used to attach an event listener to the document that fires when the user clicks anywhere on the page:
      • document.body.addEventListener("click", doSomething);
    • The click Event
      • The click event occurs when a user clicks with the mouse, presses the Enter key, or taps the screen, making it a very useful all-round event covering many types of interaction.
      • The addEventListener() method is called on a node object, the node to which the event listener is being applied.
      • document.body.addEventListener('click',doSomething);

  • The Event Object
    • Whenever an event handler is triggered by an event, the callback function is called.
    • .type returns type of event
    • .target returns refernces to node that fired event
    • Coordinated of an Event
      • screenX, screenY
      • clientX, clientY
      • pageX, pageY

  • Mouse Events
    • mousedown, mouseup
    • mouseover, mouseout
    • mousemove

  • Keyboard Events
    • keydown , keypress, keyup
    • The information about which key was pressed is a property of the event object.
      • The keydown event occurs when a key is pressed and will continue to occur if the key is held down.
      • The keypress event occurs after a keydown event but before a keyup event.
      • The keyup event occurs when a key is released.
    • Modifier Keys
      • Pressing the modifier keys such as Shift, Ctrl, Alt and meta (Cmd on Mac) will fire the keydown and keyup events, but not the keypress event as they don't produce any characters on the screen.
      • Modifier keys can often already have a purpose assigned in the browser or operating system. And although it’s possible to prevent the default behavior in the browser, it’s not considered best practice to do so.
      • shiftKey , ctrlKey , altKey , and metaKey

  • Touch Events
    • It’s important to support mouse events as well as touch events, so non-touch devices are also supported
    • touchstart
      • better to use: click event- can be thought of as a "tap" in the context of a touch event
    • touchend, touchmove, touchenter, touchleave, touchcancel
    • swipe created by using a combination of touchstart , touchmove , and touchleave events
    • Touch Event Properties
      • touches
        • also has length property
          • tells how many touch points
        • Each touch object has a number of properties, many similar to the event object
      • .identifier

  • Removing Event Listeners
    • removeEventListener()
    • Note that you shouldn't use anonymous functions as an argument to addEventListener() if you want to remove it later. This is because there needs to be a reference to the same function name in the arguments of removeEventListener() .

  • Stopping Default Behavior
    • preventDefault() is a method of the event object that can be used inside the callback function to stop the default behavior happening.

  • Event Propagation
    • Bubbling
      • event.stopPropagation()
    • Capturing

  • Event Delegation
    • can be used to attach an event listener to a parent element in order to capture events that are triggered by its child elements.
